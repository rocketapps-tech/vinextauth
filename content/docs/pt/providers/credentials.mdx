---
title: Credentials
description: Autenticação com usuário e senha usando o provider credentials.
---

O provider credentials permite tratar o login com credenciais arbitrárias — tipicamente e-mail e senha.

## Configuração

```ts title="src/auth.ts"
import Credentials from 'vinextauth/providers/credentials';

VinextAuth({
  providers: [
    Credentials({
      credentials: {
        email: { label: 'E-mail', type: 'email' },
        password: { label: 'Senha', type: 'password' },
      },
      async authorize(credentials) {
        const user = await db.user.findUnique({
          where: { email: credentials.email },
        });

        if (!user || !verifyPassword(credentials.password, user.passwordHash)) {
          return null; // retorne null para negar o login
        }

        return { id: user.id, email: user.email, name: user.name };
      },
    }),
  ],
});
```

## Token CSRF

O provider credentials requer um token CSRF válido em cada POST. Isso é tratado automaticamente pela página de login embutida do VinextAuth.

Se você criar um formulário de login customizado, busque o token primeiro:

```ts
const { csrfToken } = await fetch('/api/auth/csrf').then((r) => r.json());
```

Depois inclua como campo oculto:

```html
<input type="hidden" name="csrfToken" value="{csrfToken}" />
```

## Rate Limiting

O login com credentials tem rate limiting embutido (5 tentativas por 15 minutos por IP). Configure via `credentials.rateLimit`:

```ts
VinextAuth({
  credentials: {
    rateLimit: {
      maxAttempts: 5,
      windowMs: 15 * 60 * 1000, // 15 minutos
    },
  },
});
```

Para ambientes distribuídos (múltiplos Workers), use o [adaptador KV Rate Limiter](/pt/docs/adapters/cloudflare-kv-rate-limiter).
