# VinextAuth — Claude Code Guide

## Project Overview

**VinextAuth** is a drop-in replacement for NextAuth v4, built specifically for the [Vinext](https://vinext.io/) framework and Cloudflare Workers edge runtime. It has zero Node.js dependencies and uses the pure Web Crypto API for all cryptographic operations.

- **Package name:** `vinext-auth`
- **Version:** 0.3.x
- **License:** MIT
- **Website:** https://vinext.io/

## Architecture Principles

1. **Edge-first**: Every code path must run in Cloudflare Workers / Vinext edge runtime. Never use Node.js APIs (`crypto`, `fs`, `process`, etc.) — always use the Web Crypto API (`globalThis.crypto`).
2. **Zero dependencies**: The published library has no runtime `dependencies`. All crypto, JWT, and cookie logic is hand-rolled using platform APIs.
3. **NextAuth v4 API compatibility**: Public API surface mirrors NextAuth v4 as closely as possible to be a drop-in replacement. Preserve method signatures, callback names, and hook behavior.
4. **Type safety**: Strict TypeScript. Generics are used for custom session/user extension (see `VinextAuthConfig<U, S>`).
5. **Pure functions where possible**: Handlers receive a `Request` and return a `Response`. Side effects (cookies, DB) are explicit.

## Key Design Decisions

### JWT-first Sessions
Sessions are stored as signed JWTs in cookies by default. The Cloudflare KV adapter enables database sessions. JWT signing uses HMAC-SHA256 derived from `NEXTAUTH_SECRET`.

### CSRF Protection
The credentials provider requires a valid CSRF token on POST. CSRF tokens are double-submit cookies signed with the secret.

### Rate Limiting (Credentials)
Built-in in-memory rate limiter for credentials sign-in. Configurable via `rateLimit` option. Resets after window expiry.

### Token Refresh Lock
Prevents race conditions during JWT refresh by using a shared lock primitive (`refresh-lock.ts`).

## Monorepo Workspaces

The `apps/` directory uses npm workspaces. Apps reference the library as:

```json
"vinextauth": "*"
```

npm resolves this to the workspace root package. Both apps use **Vinext** — Cloudflare's Vite-powered framework compatible with the NextAuth API surface.

Run apps from root:

```bash
npm run dev:vinext          # apps/dev/vinext on port 3001
npm run dev:example-vinext  # apps/examples/vinext-basic on port 3002
```

Or install + run directly:

```bash
cd apps/dev/vinext
cp .env.example .env.local   # fill in secrets
npm install
npm run dev      # runs: vinext dev --port 3001
npm run deploy   # deploys to Cloudflare Workers via: vinext deploy
```

## Build System

- **Tool:** [tsup](https://tsup.egoist.dev/) — wraps esbuild + tsc
- **Output:** `dist/` — ESM only (`.js` + `.d.ts` per entry point)
- **Entries:** Defined in `tsup.config.ts` and mirrored in `package.json#exports`

```bash
npm run build       # one-time build
npm run dev         # watch mode (for local development with apps/)
```

## Testing

```bash
npm run test          # vitest run (single pass)
npm run test:watch    # vitest watch
npm run typecheck     # tsc --noEmit (no emit, checks types only)
```

Tests live alongside source in `src/` or in a `tests/` directory. Use `vitest` globals — no import needed for `describe`, `it`, `expect`.

## Environment Variables

| Variable | Required | Description |
|---|---|---|
| `NEXTAUTH_SECRET` | Yes | Base64 secret for JWT signing. Generate: `openssl rand -base64 32` |
| `NEXTAUTH_URL` | Yes | Canonical URL of the app (e.g. `https://app.vinext.io`) |
| `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET` | OAuth | Google provider credentials |
| `GITHUB_CLIENT_ID` / `GITHUB_CLIENT_SECRET` | OAuth | GitHub provider credentials |

## Common Tasks

### Add a new OAuth provider
1. Create `src/providers/<name>.ts` — export a function returning `OAuthProvider` config
2. Add export to `src/index.ts` (or as a sub-export)
3. Add entry to `tsup.config.ts` and `package.json#exports`

### Add a new session adapter
1. Create `src/adapters/<name>.ts` — implement the `SessionAdapter` interface from `types.ts`
2. Add export to `package.json#exports` under `./adapters/<name>`

### Modify the auth handler flow
- Entry point: `src/handlers/index.ts` — routes by pathname
- Each handler receives `(req: Request, config: NormalizedConfig): Promise<Response>`

## Code Style

- TypeScript strict mode — no `any`, no `@ts-ignore` without comment
- ESM modules — `import/export`, no `require()`
- Functions over classes for handlers
- Named exports everywhere; default export only for the main `VinextAuth()` factory
- No trailing `console.log` in library code

## Releases

Versioning follows semver. Keep `NEXTAUTH_SECRET`-compatible JWT format stable across minor versions (breaking change = major).
